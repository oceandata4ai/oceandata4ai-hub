Get Started with seekdb: Install, Deploy, and Run Your First Hybrid Search

OceanBase User Group · Mar 6, 2026 · 8 min read

If you're building a RAG application, you've probably stitched together a vector database, a search engine, and an application database. seekdb aims to collapse that stack into one system — and this guide gets you from zero to a working hybrid search in about 15 minutes.

What you'll need

Docker (recommended for local dev), or a Linux/macOS host for native install. Basic familiarity with SQL helps but isn't required — we'll use the CLI for everything.

Quick install

docker pull oceanbase/seekdb:latest
docker run -d --name seekdb -p 2881:2881 oceanbase/seekdb:latest

Once the container is healthy, connect with the default credentials from the docs and create your first table with a vector column.

Your first hybrid query

Hybrid search combines dense vector similarity with keyword matching — the pattern most production RAG systems eventually need. With seekdb, both run against the same table:

SELECT id, title, score
FROM hybrid_search(
  'machine learning tutorials',
  embedding := query_embedding('how to train a model'),
  limit := 10
);

That's it — one query, one database. No syncing embeddings to a separate vector store.
